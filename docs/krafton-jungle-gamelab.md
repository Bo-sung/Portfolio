## 왜 게임랩인가

- **혼자서는 60~70%에서 멈추는 패턴을 알고 있습니다.** Skul 모작, MechTarkov(2년째 진행 중), 러닝게임 — 기능이 대부분 돌아가는 시점에서 폴리싱과 마무리 대신 다음 프로젝트로 넘어가곤 했습니다. 저장소마다 그 흔적이 정직하게 남아 있습니다.
- 반대로 **마감과 동료가 있는 환경에서는 끝까지 갔습니다.** 1박 2일 게임잼 우승, 팀 리드로 기한 내 완성, 실무 2년 3개월의 출시·운영 — 제 완주 이력은 전부 "함께 만든" 프로젝트에서 나왔습니다.
- 그래서 게임랩입니다. 다양한 사람들과 부딪히며 협업하는 기회 자체가 지금 제게 가장 부족하고, 가장 필요한 경험입니다. 클라이언트(2년 3개월)·서버(8개월) 실무와 기획 문서 작성까지 게임 개발 전반의 시야를 만들어 온 만큼, 팀 안에서 어느 자리든 메울 수 있는 상태로 들어갑니다. 최종 목표는 **게임 개발 전 영역을 이해하는 디렉터**입니다.

## 완주 이력 — 전부 "함께 만든" 프로젝트에서 나왔습니다

### 🏆 무박 2일 게임잼 우승 — Creavas (2025.06.28~29, 4인 팀)

- "얼음" 주제, **참가팀 중 유일하게 Unreal Engine(5.6 + Blueprint)을 선택한 팀**. 빙벽을 오르며 **체온(체력 100)이 소진되기 전에 목표 높이에 도달**하는 생존 게임 — 핫팩(체온 손실 방지)·피켈(붙잡기 포인트 생성) 아이템으로 리스크를 관리하는 구조
- **결정적 피봇**: 초기 기획은 단순 플랫포머였지만, 빠른 프로토타이핑 결과 "너무 단순하다"는 결론 → **'Getting Over It' 방식 발사형 이동으로 방향 전환**하자 게임성이 획기적으로 개선됨. *명확한 방향성이 결과의 질을 좌우한다*는 교훈을 몸으로 얻은 순간
- 담당: 게임 시스템 개발(아이템·체온 시스템, 난이도 구성)
- **잼이 끝난 뒤에도 멈추지 않음**: 1회성으로 끝내기 아까워 확장성을 고려한 **데이터 테이블과 엑셀 기반 레벨 에디터**를 추가 구현 — 기획자가 원하는 맵을 직접 만들 수 있는 구조로 발전시킴

### 개인 프로젝트 상세 — 완성한 것과 멈춘 것 모두

**BattleShip (C#/.NET 10, 완성)** — [저장소](https://github.com/Bo-sung/BattleShip_Server)
- HTTP 없이 6바이트 length-prefix 바이너리 TCP 프로토콜을 직접 설계한 3단 분산 서버
- Auth(MySQL + Redis 1회용 토큰) → Lobby(방 매칭) → GameSession **독립 프로세스 동적 spawn**(포트 풀 7010~7200)
- Watchdog 좀비 프로세스 정리, 결과 ACK 3회 재시도, 게임 룰 JSON 외부화, SSH 배포 스크립트까지 — 혼자서도 "작게 끝까지"는 가능함을 증명한 프로젝트

**InterplanetaryOnline (팀 4인, 서버 메인, 완성)** — [저장소](https://github.com/Bo-sung/InterPlanetery_Server)
- 락스텝 결정론적 동기화 RTS 서버 전체 설계·구현 — 입력 버퍼링·지연 보상으로 동기화 안정화
- 추상화된 네트워크 매니저 API로 팀원들이 네트워크를 몰라도 게임 로직 구현 가능하게 지원 (동접 8명 테스트)

**MechTarkov (UE5, 2년+ 진행 중 — 60~70%의 대표 사례)**
- 2040년대 대구·경북 배경 포스트 아포칼립스 **익스트랙션 슈터 × 메카**. Obsidian으로 장르 분석·세계관·프로젝트 규칙 문서화
- 2023 프로토타입 → 2024 Lyra 기반 클라이언트(119커밋) → 2025 GAS 기반 C++ 재구현 → 에셋 통합(60GB)
- 가장 오래, 가장 진지하게 붙잡고 있는 프로젝트이자 — 혼자서는 완성 못 하는 한계를 가장 크게 체감시킨 프로젝트

**WPF 마리오 (2021, 1-1 스테이지 완성)**
- 엔진 없이 AABB 충돌, 스윕 방식 터널링 해결, 겹침 위치 보정, 15ms 게임 루프(Physics→Update)를 직접 구현
- 4년 뒤 락스텝 서버를 만들 때의 밑바탕이 된 기본기 훈련

**ASPAuthServer + authLobbyGame (C# 서버 독학 연작, 2025~)** — [저장소](https://github.com/Bo-sung/ASPAuthServer)
- ASP.NET Core 인증 서버(RESTful, MySQL+Redis, 계정 잠금 정책) + WPF GM 툴(토큰 자동 갱신, 실시간 로그 뷰어) 세트 구현

## AI를 개발력으로 바꾼 과정

AI를 "써보는" 단계를 지나 **파이프라인과 도구를 만드는 단계**까지 왔습니다. 유료 구독을 유지하며 활용법을 계속 탐구 중입니다.

**1) 아트 파이프라인 자동화 — 프로그래머 혼자서도 비주얼을 만들기 위해**
- [SpriteForge](https://github.com/Bo-sung/SpriteForge): 리깅된 3D 모델(FBX/GLB)을 **픽셀아트 스프라이트 시트로 자동 변환** — OpenGL 오프스크린 렌더링 → 대표색 다운스케일 → 팔레트 양자화 → 시트 패킹. CLI + 라이브 프리뷰 GUI
- [ComfyUI-FBX-ControlNet-Converter](https://github.com/Bo-sung/ComfyUI-FBX-ControlNet-Converter): 3D 애니메이션을 ComfyUI 안에서 **ControlNet 패스(OpenPose/depth/normal/canny)로 변환**하는 커스텀 노드 — 실제 스켈레톤 기반이라 프레임 간 흔들림 없는 AI 영상 생성 가능. C#/OpenGL 네이티브 렌더러 내장, Git LFS 배포
- [html_to_ugui](https://github.com/Bo-sung/html_to_ugui): AI로 뽑은 HTML/CSS UI 시안을 헤드리스 Chromium으로 렌더링해 **Unity UGUI 프리팹으로 자동 변환** — 반복 구조는 재사용 프리팹으로 자동 추출

**2) AI 에이전트 오케스트레이션 — 개발 프로세스 자체를 바꾸기**
- [AgentManager](https://github.com/Bo-sung/AgentManager): 여러 AI 코딩 에이전트(Claude Code, Codex 등)를 **git worktree로 격리해 병렬 관리**하는 .NET 10 WPF 플랫폼. 로컬 LLM(Ollama) 번역 레이어로 클라우드 토큰 절감, Git Diff 리뷰 페인, 대형→소형 모델 워커 위임 구조
- [ai-workspace-template](https://github.com/Bo-sung/ai-workspace-template): 여러 AI 워커가 한 저장소에서 협업할 때의 **역할·권한 거버넌스 규칙**을 템플릿화 — 오케스트레이터만 Git 쓰기 가능, 워커는 읽기 전용·토큰 제한

**3) 게임 안에 AI 서비스 통합**
- 캡스톤 "폴리글롯": Papago 번역 + ML Kit 온디바이스 폴백 + ReturnZero/Google STT 이중화 — 외부 AI 서비스를 안정적으로 묶는 설계

게임랩에서도 이 도구들이 그대로 팀의 생산성이 됩니다 — 아트 리소스가 부족한 프로그래머 팀에서 AI 파이프라인으로 비주얼 공백을 메우고, 반복 작업을 자동화하는 역할을 할 수 있습니다.
