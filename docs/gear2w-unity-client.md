## 2. 콘텐츠 파이프라인·SDK 대응 이력 (실무)

**"콘텐츠를 빠르게, 반복 가능하게"를 실무에서 도구와 구조로 풀어 온 이력입니다.**

- **이벤트 시스템 테이블화**: 이벤트마다 코드를 새로 짜던 구조를 데이터 테이블 기반으로 개편 — 이벤트 추가 프로세스 간소화
- **기획자용 엑셀 → 게임 테이블 자동 추출 툴 제작**: 기획자가 개발자를 거치지 않고 데이터를 반영하는 흐름을 만듦
- **SDK 의존부 추상화·교체**: 3인 신규 프로젝트에서 기존 코드의 SDK 의존부를 인터페이스로 추상화한 뒤 **NHN GAMEBASE, IronSource** 등으로 교체 — 광고/플랫폼 SDK가 바뀌어도 게임 코드가 흔들리지 않는 구조
- 게임잼(학생 주도, 우승)에서도 같은 접근: 잼 도중 데이터 테이블 + 엑셀 기반 레벨 에디터를 구현해 스테이지를 빠르게 양산

## 5. AI 파이프라인 툴 연작 (개인, 전부 공개)

아트 리소스 병목을 AI로 풀기 위해 직접 시도·연구하며, 체감한 한계를 메우는 과정에서 만든 도구들입니다.

| 툴 | 기능 |
| --- | --- |
| [SpriteForge](https://github.com/Bo-sung/SpriteForge) | 리깅된 3D 모델(FBX/GLB) → **픽셀아트 스프라이트 시트** 자동 변환 — 프레임 간 일관성을 구조적으로 보장 |
| [ComfyUI-FBX-ControlNet-Converter](https://github.com/Bo-sung/ComfyUI-FBX-ControlNet-Converter) | FBX 애니메이션 → **ControlNet 패스** 변환 — 시간적으로 안정된 AI 영상 생성 |
| [html_to_ugui](https://github.com/Bo-sung/html_to_ugui) | AI로 뽑은 HTML/CSS UI 시안 → **Unity UGUI 프리팹** 자동 변환 |
| [AgentManager](https://github.com/Bo-sung/AgentManager) | AI 코딩 에이전트 다중 운용 플랫폼 (git worktree 격리, Diff 리뷰) |

주요 AI 서비스를 종류별로 유료 구독해 비교하고, 로컬 LLM용 장비까지 도입해 검증했습니다 — **AI를 콘텐츠 생산 속도로 바꾸는 방법**을 계속 실험 중입니다.
